API Reference

Configuration Interface 

interface GaugeChartConfig {
  value: number;
  min?: number;
  max?: number;
  width?: number;
  height?: number;
  animate?: boolean;
  colors?: string[];
  label?: string;
  unit?: string;
}

Data Structure 

interface GaugeData {
  value: number;
  min: number;
  max: number;
  label?: string;
  unit?: string;
}

Properties Reference

Property Type Default Description
value number Required Current value to display on the gauge
min number 0 Minimum value for the gauge
max number 100 Maximum value for the gauge
animate boolean true Enable or disable gauge animations
label string '' Label to display on the gauge

Installation & Setup

Step 1: Install the Library 

npm install @synerity/charts

Add the chart library to your Angular project dependencies.

Step 2: Import the Component 

import { GaugeChartComponent } from '@synerity/charts';

@NgModule({
  imports: [GaugeChartComponent],
  // ...
})
export class AppModule { }

Import the GaugeChartComponent in your module or use it as a standalone component.

Step 3: Prepare Your Data 

const gaugeData = {
  value: 75,
  min: 0,
  max: 100,
  label: 'Performance',
  unit: '%'
};

Format your data as an object with value, min, max, label, and optional unit properties.

Implementation Examples

Basic Implementation 

// Component TypeScript
export class MyComponent {
  gaugeConfig: GaugeChartConfig = {
    data: {
      value: 75,
      min: 0,
      max: 100,
      label: 'CPU Usage',
      unit: '%'
    },
    width: 300,
    height: 300
  };
}

// Template HTML
<app-gauge-chart 
  [config]="gaugeConfig"
  title="System Performance"
  description="Current CPU utilization">
</app-gauge-chart>

Advanced Implementation 

// Component TypeScript
export class MyComponent {
  gaugeConfig: GaugeChartConfig = {
    data: this.getGaugeData(),
    width: 400,
    height: 400,
    animate: true,
    colors: ['#EF4444', '#F59E0B', '#10B981'],
    showValue: true,
    showLabel: true
  };

  onGaugeClick(event: any) {
    console.log('Gauge clicked:', event);
  }

  onValueChange(event: any) {
    console.log('Value changed:', event);
  }
}

Best Practices & Tips

Value Ranges

Set appropriate min and max values that reflect the actual range of your data for meaningful visualization.

Color Coding

Use color schemes that intuitively represent performance levels (red for critical, yellow for warning, green for good).

Real-time Updates

Consider implementing real-time data updates for dynamic gauge displays in monitoring applications.

Accessibility

Provide meaningful labels, descriptions, and ARIA labels for screen readers. Ensure sufficient color contrast.

Error Handling

Always handle potential errors gracefully, such as invalid values, missing data, or out-of-range values.

Interactive Features

Leverage click handlers and hover effects to provide additional context and user interaction.
