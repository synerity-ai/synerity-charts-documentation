Installation & Setup

Step 1: Install the Library 

npm install @synerity/charts

Add the chart library to your Angular project dependencies.

Step 2: Import the Component 

import { TreemapChartComponent } from '@synerity/charts';

@NgModule({
  imports: [TreemapChartComponent],
  // ...
})
export class AppModule { }

Import the TreemapChartComponent in your module or use it as a standalone component.

Step 3: Prepare Your Data 

const treemapData = [
  { name: 'Category A', value: 100, children: [
    { name: 'Sub A1', value: 60 },
    { name: 'Sub A2', value: 40 }
  ] },
  { name: 'Category B', value: 80, children: [
    { name: 'Sub B1', value: 50 },
    { name: 'Sub B2', value: 30 }
  ] }
];

Format your data as a hierarchical structure with name, value, and optional children properties.

Implementation Examples

Basic Implementation 

// Component TypeScript
export class MyComponent {
  treemapConfig: TreemapChartConfig = {
    data: [
      { name: 'Sales', value: 1000, children: [
        { name: 'Q1', value: 250 },
        { name: 'Q2', value: 300 },
        { name: 'Q3', value: 200 },
        { name: 'Q4', value: 250 }
      ] }
    ],
    width: 600,
    height: 400
  };
}

// Template HTML
<app-treemap-chart 
  [config]="treemapConfig"
  title="Sales Distribution"
  description="Quarterly sales breakdown">
</app-treemap-chart>

Advanced Implementation 

// Component TypeScript
export class MyComponent {
  treemapConfig: TreemapChartConfig = {
    data: this.generateTreemapData(),
    width: 800,
    height: 500,
    animate: true,
    showLabels: true,
    showValues: true,
    colorScheme: 'category10',
    padding: 2
  };

  onNodeClick(event: any) {
    console.log('Node clicked:', event);
  }

  onNodeHover(event: any) {
    console.log('Node hovered:', event);
  }
}

Best Practices & Tips

Data Hierarchy

Organize your data in a logical hierarchical structure with meaningful category names and values.

Color Schemes

Use consistent color schemes that help distinguish between different categories and hierarchy levels.

Performance Optimization

Limit the depth and number of nodes for complex hierarchies to maintain smooth performance.

Accessibility

Provide meaningful titles, descriptions, and tooltips for screen readers. Ensure sufficient color contrast.

Error Handling

Always handle potential errors gracefully, such as empty data, invalid values, or missing properties.

Interactive Features

Leverage hover effects and click handlers to provide additional context and navigation.
