API Reference

Configuration Interface 

interface PieChartConfig {
  data: ChartData[];
  width?: number;
  height?: number;
  animate?: boolean;
  showLabels?: boolean;
  showLegend?: boolean;
  innerRadius?: number;
  outerRadius?: number;
  colors?: string[];
}

Data Structure 

interface ChartData {
  label: string;
  value: number;
  color?: string;
}

Properties Reference

Property Type Default Description
data ChartData[] Required Array of data points to display in the chart
width number 600 Width of the chart in pixels
height number 400 Height of the chart in pixels
animate boolean true Enable or disable chart animations
showLabels boolean true Show or hide data labels on slices
showLegend boolean true Show or hide the legend
innerRadius number 0 Inner radius for donut chart (0 for pie chart)
outerRadius number 200 Outer radius of the chart

Installation & Setup

Step 1: Install the Library 

npm install @synerity/charts

Add the chart library to your Angular project dependencies.

Step 2: Import the Component 

import { PieChartComponent } from '@synerity/charts';

@NgModule({
  imports: [PieChartComponent],
  // ...
})
export class AppModule { }

Import the PieChartComponent in your module or use it as a standalone component.

Step 3: Prepare Your Data 

const pieData = [
  { label: 'Sales', value: 120, color: '#3B82F6' },
  { label: 'Marketing', value: 80, color: '#10B981' },
  { label: 'Development', value: 100, color: '#F59E0B' }
];

Format your data as an array of objects with label, value, and optional color properties.

Implementation Examples

Basic Implementation 

// Component TypeScript
export class MyComponent {
  pieConfig: PieChartConfig = {
    data: [
      { label: 'Sales', value: 120 },
      { label: 'Marketing', value: 80 },
      { label: 'Development', value: 100 }
    ],
    width: 400,
    height: 400
  };
}

// Template HTML
<app-pie-chart 
  [config]="pieConfig"
  title="Revenue Distribution"
  description="Revenue breakdown by department">
</app-pie-chart>

Advanced Implementation 

// Component TypeScript
export class MyComponent {
  pieConfig: PieChartConfig = {
    data: this.generatePieData(),
    width: 500,
    height: 500,
    animate: true,
    showLabels: true,
    showLegend: true,
    innerRadius: 50,
    outerRadius: 150
  };

  onSliceClick(event: any) {
    console.log('Slice clicked:', event);
  }

  onSliceHover(event: any) {
    console.log('Slice hovered:', event);
  }
}

Best Practices & Tips

Segment Limits

Limit to 5-7 segments for optimal readability and avoid overwhelming users with too many categories.

Color Schemes

Use distinct, accessible colors for each segment and ensure good contrast for readability.

Label Positioning

Use clear, concise labels and consider using a legend for better organization of segment information.

Accessibility

Provide meaningful titles, descriptions, and ARIA labels for screen readers. Ensure sufficient color contrast.

Error Handling

Always handle potential errors gracefully, such as empty data, invalid values, or missing properties.

Interactive Features

Leverage hover effects and click handlers to provide additional context and user interaction.
